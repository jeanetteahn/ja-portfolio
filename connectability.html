ConnectAbility

Bridging communities through accessible events—a centralized platform to help individuals with disabilities, caregivers, and families find inclusive community resources in Washtenaw County.

Role UX Designer & Researcher
Project Type Academic/UX Course Project
Timeline August - December 2024
Tools Figma, Miro, User Testing, Storyboarding

Project Brief

ConnectAbility is a UX design course project inspired by my personal experiences growing up in Washtenaw County, Michigan. The platform addresses a critical gap in how individuals with disabilities, caregivers, and families discover inclusive community events and resources.

The idea emerged from a personal memory of watching a family member struggle to organize resources for a loved one with a disability. That experience shaped both the problem I pursued and the solution I created, allowing me to connect real-world empathy with formal UX methods.

Personal Motivation

Growing up in Washtenaw County, I witnessed firsthand how fragmented community resources can create barriers for families seeking inclusive activities. This project represents my commitment to using design as a tool for meaningful social impact.

Research & Problem Definition

Problem Statement

How might we create a centralized, accessible platform in Washtenaw County to streamline event discovery for people with disabilities, caregivers, and their families?

Research Insights

~20%

Only about 20% of individuals with disabilities in Washtenaw County report receiving adequate care

📄

Fragmented Resources: Community resources are scattered across flyers, websites, and word-of-mouth communication

⚡

Increased Workload: This fragmentation creates additional barriers to participation and increases cognitive load

Understanding the Landscape

Through initial research, I discovered that families and caregivers often spend significant time piecing together information from multiple sources. This not only creates frustration but also means many miss out on valuable community connections and resources.

Design Process

Storyboarding & Ideation

I explored multiple formats (website vs. mobile app) and visualized user scenarios through storyboards. This process clarified how different touchpoints could address accessibility needs and user priorities.

User storyboards exploring different scenarios and platform formats

Personas & Journey Mapping

Developed three key personas capturing different user perspectives:

  • Person with a disability - Direct user seeking accessible events
  • Caregiver - Professional seeking resources for clients
  • Family member - Parent or relative coordinating activities

Used Miro to map user journeys with color-coded Post-its, synthesizing common needs such as simplified search, trustworthy content, and reduced cognitive load.

User personas and journey mapping process using Miro with color-coded insights

Prototyping & Testing

My iterative approach included:

  • Started with paper sketches to explore layout concepts
  • Moved to low-fidelity digital wireframes in Figma
  • Conducted usability testing with family members of individuals with disabilities

🔍 Key Testing Insight

Users wanted reassurance about credibility and trustworthiness. This feedback directly led to adding an About page to communicate the platform's mission and build user confidence.

Final Prototype Features

The final ConnectAbility prototype focuses on simplicity, accessibility, and trust-building to create an inclusive experience for all users.

🔍

Smart Search & Filtering

Search and filter events by category and specific accessibility needs

📅

Multiple View Options

Calendar and map views support different cognitive styles and preferences

🔐

Simple Authentication

Streamlined login flow (name + password) reduces barriers for less tech-comfortable users

🧭

Consistent Navigation

Clear navigation with Home, About, Calendar, and Map sections

♿

WCAG-Informed Design

Inclusive layout, color contrast, and interactions following accessibility standards

🤝

Trust-Building Elements

Dedicated About page communicates mission and builds user confidence

Key screens highlighting search functionality, calendar view, and accessibility features

Key Takeaways

This project taught me how to balance rapid ideation with strategic prioritization while keeping accessibility and user empathy at the center of every design decision.

🎯

Scoping & Prioritization

Learned how to balance rapid ideation with selecting core features when working independently. Understanding what to build first vs. what to save for future iterations.

♿

Accessibility in Practice

Applying WCAG standards showed how small design details—like color contrast or navigation consistency—directly shape inclusivity and user experience.

🔄

Iterative Design Value

Continuous feedback shaped meaningful features such as the About page, reinforcing the importance of testing with real users throughout the design process.

❤️

Empathy-Driven Design

Personal experience informed the project direction, but formal UX methods validated and refined the solution to serve broader user needs.

Impact & Future Vision

Although created as a class project, ConnectAbility demonstrates how I applied end-to-end UX methods—from research and personas to prototyping and testing—to design an inclusive solution grounded in empathy.

By centralizing resources into one platform, the concept reduces the stress of searching for events and helps families spend more time participating in their communities.

Next Steps

If further developed, ConnectAbility would expand to include:

📚 Event History & Reminders

Booked event history and personalized reminder system

💬 Community Forums

Discussion spaces for user interaction and community building

👤 Enhanced Profiles

Profile customization and Google sign-in integration

🔗 Comprehensive Support

Footer with FAQs, Help, Terms of Service, and social links
